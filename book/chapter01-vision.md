# Chapter 1: Vision

## ビジョン

> **「AIエージェントが、データの意味を理解し、メタデータを自律的に管理する世界」**

エンタープライズのデータ資産管理は、長年にわたり人手に頼ってきました。スキーマの登録、データカタログの更新、品質ルールの設定――これらは専門知識を要する反復作業であり、データチームのボトルネックになっています。

本プラットフォームは、AI エージェントがその作業を担うことで、データエンジニアが本質的な価値創出に集中できる環境を提供します。

## 解決する問題

### 現状の課題

```
【Before】
開発者がDBスキーマを変更
    ↓
手動でOpenMetadataに登録（1-2時間）
    ↓
ドキュメントを別途更新（30分）
    ↓
データ品質ルールを手動設定（1時間）
    ↓
レビュー待ち（1-2日）
合計: 3-4時間 + 待機時間
```

### あるべき姿

```
【After】
開発者がDBスキーマを変更
    ↓
AI Agentが自動検出・登録（5分）
    ↓
メタデータ・品質ルール自動生成（自動）
    ↓
Slackに通知・承認フロー開始（自動）
合計: 5分（人間の作業ゼロ）
```

## 設計原則

### 1. Agent-First
人間が介入するのは「判断」だけ。定型作業はすべてエージェントが実行する。

### 2. Tool-First
エージェントの能力は Tool の集合で定義される。Tool は OpenMetadata API / Quarkus API をラップした再利用可能な関数群。

### 3. Metadata-First
すべてのビジネスエンティティは OpenMetadata に登録され、AI が検索・参照できる状態を維持する。

### 4. Human-in-the-Loop
重要な変更（本番 DB スキーマ変更、データ削除等）は必ず人間の承認を経る。
